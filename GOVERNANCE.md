# Governance

QuantGlass uses maintainer-led governance while the project is young.

## Maintainers

Maintainers are responsible for:

- Project direction and release decisions.
- Reviewing pull requests.
- Protecting the licensing model.
- Enforcing the code of conduct.
- Handling security reports.
- Deciding what belongs in the community edition versus commercial offerings.

## Decision Making

Most decisions are made through GitHub issues and pull requests. For larger
changes, open an issue or discussion before implementation.

Large changes include:

- New broker or live-trading flows.
- Strategy engine architecture changes.
- Persistent schema changes.
- Licensing or contributor-policy changes.
- Cloud, sync, hosted, or telemetry features.

## Community Roles

- **Contributor:** submits issues, docs, tests, code, or reviews.
- **Regular contributor:** repeatedly contributes useful work in a focused area.
- **Maintainer:** has commit/release authority and responsibility for project
  direction.

Maintainer status is granted by existing maintainers after sustained,
trustworthy contribution.

## Commercial Boundary

The AGPL Community Edition stays useful and extensible. Commercial offerings may
fund signed releases, support, proprietary embedding, enterprise integrations,
and hosted/private deployments.
